## Proposed Merge Groups

### 1. Network Utilities → merge into `/network` (new tab)

**Tabs:** Network Tools · DNS Lookup · DNS Trace · Port Scanner · Wake-on-LAN

**Rationale:** `/dns`, `/dns-trace`, `/ports`, and `/wol` are all single-form
query tools ("enter host → run → read result"). The existing `/network` page
already has multiple internal tab sections. Absorbing these into a unified
Network Tools page reduces 5 sidebar entries to 1.

**Risk:** Low — all tabs are independent, no shared state needed.

---

### 2. System Health → new `/infrastructure` page

**Tabs:** Services · Docker · Databases · Benchmarks

**Rationale:** All four pages monitor running infrastructure on the host.
Services, Docker, and Databases share identical card-grid UI patterns.
A user debugging a production issue visits all four in sequence.

**Risk:** Medium — Docker and Services pages are moderately complex; tabs
must not lose their individual scroll position.

---

### 3. Personal Tools → new `/tools` page

**Tabs:** Clipboard · QR Code · Speed Test · Screenshots

**Rationale:** All four are lightweight, standalone utility widgets with
minimal UI (one action → one output). None is visited frequently enough
to warrant its own sidebar slot.

**Risk:** Low — completely independent tools, no shared state.

---

### 4. App Diagnostics → new `/diagnostics` page

**Tabs:** Status · Logs · Notifications · Internals · Docs · Showcase

**Rationale:** All are read-only views of application state. A developer
investigating a problem visits them in sequence. `Showcase` (component
demo) fits here as a dev-tools tab.

**Risk:** Low — all read-only, no mutations needed across tabs.
